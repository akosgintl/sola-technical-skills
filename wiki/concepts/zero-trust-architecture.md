# Zero-Trust Architecture

> [!summary]
> A security model that trusts no request implicitly — verifying identity, device, and context on every access and enforcing least privilege regardless of network location.

**Priority:** 🟠 P1 · **Domain:** [[tier-1-edge|Security & Compliance]] · **Roadmap:** §3.1.1

## What it is

Zero-trust architecture replaces perimeter-based security ("inside is trusted") with continuous verification of every request. Each access decision considers identity, device posture, and context, grants the minimum privilege needed, and assumes breach so blast radius is contained. Microsegmentation and strong identity are foundational, and it now extends to non-human principals like services and agents.

## Key concepts

- Never trust, always verify
- Least-privilege and just-in-time access
- Microsegmentation
- Continuous / context-aware authentication
- Assume-breach posture
