# LLM Application Architecture

> [!summary]
> The end-to-end stack for building LLM-powered applications — model access, retrieval, context assembly, memory, orchestration, and guardrails — and the design choices that connect them.

**Domain:** [[tier-1-edge|AI & Agentic Architecture]]

## What it is

LLM application architecture is the reference structure of a production LLM system: how prompts and context are assembled, how external knowledge is retrieved, how models are selected and called, how memory persists across turns, and how outputs are validated. It treats the LLM as one component in a larger system rather than the whole application, emphasizing the surrounding plumbing that makes it reliable.

## Key concepts

- Retrieval-augmented generation
- Context engineering and window budgeting
- Memory architectures
- Model selection and routing
- Guardrails and output validation
